现状诊断
当前完全没有「项目」概念，搜索走的是 ViewModel 里硬编码的 mock 数据。文件系统零接触。所以三件事要一起想清楚再动手：存哪、怎么建/导入、搜索怎么进化。

一、存哪
Android 上有三条路：

方案	路径	权限	用户能看到
App 内部私有目录	filesDir/projects/	无需	❌ 卸载即消失
App 外部私有目录	getExternalFilesDir("projects")	无需	✅ Files app 可见
SAF 任意文件夹	用户用 picker 选，存 Uri	一次授权永久	✅ 原地访问
我的建议：Hybrid

新建项目  →  getExternalFilesDir("projects")/[name]/   不需要权限，用户在 Files 里看得到
导入项目  →  ACTION_OPEN_DOCUMENT_TREE (SAF)，persisted Uri 存 Room

minSdk 36 (Android 16)，SAF 很稳，持久化 Uri takePersistableUriPermission() 重启后依然有效。内部存项目元数据用 Room，实际文件跟着各自的路径走。

二、项目模型长啥样
Project
  ├── id, name, description?
  ├── rootPath / rootUri (internal: 绝对路径; external: SAF Uri string)
  ├── isExternal: Boolean
  ├── language: CodeLanguage?   ← 自动检测
  ├── lastOpened: Long
  ├── isPinned: Boolean
  └── gitRemote: String?        ← 留给以后

自动检测项目类型是我觉得最值得做的一个细节：扫根目录一层，build.gradle.kts → Android，package.json → Node，requirements.txt → Python，自动打上语言标签、设置好图标颜色，开箱即用的感觉会非常好。

三、搜索怎么进化（这是最有趣的部分）
现在搜索栏是「全局命令面板」，以后它应该根据上下文自动切换范围：

无项目打开（主页）
  ├── 无前缀  →  搜索项目名 + 跨项目最近文件
  ├── >       →  全局命令
  └── @       →  按项目名过滤（新前缀）
项目内
  ├── 无前缀  →  当前项目内文件
  ├── #       →  当前项目内符号/函数
  └── >       →  编辑器命令（保存、格式化、git...）

我特别喜欢一个想法：主页的 Wing chips（左右两侧那些小 chip）现在显示的是最近文件，可以改成显示最近项目——点一下直接跳入，像是常驻在搜索栏两侧的项目快捷入口，跟整体的「Command Stage」美学完全吻合。

四、建/导入的 UX
这是最需要想清楚的交互节点。几个方向：

A — 命令走命令面板 > New Project → 弹出一个 bottom sheet，填名字、选模板（空白/Kotlin/Python/Node），自动初始化目录结构。 > Open Folder → 调起系统 DocumentTree Picker，授权后直接进项目。

B — 主页卡片式项目列表 主页闲置时不只是那个大 greeting，下面还有最近项目的卡片。每张卡显示语言颜色条、项目名、最近文件数。长按 → pin/删除。

C（奇思妙想）— 拖拽导入 Android 13+ 支持跨 app 拖拽。从 Files app 把文件夹拖到 Axiom，直接导入。这个做出来演示效果极好，虽然用户不一定常用。

五、一个更大的结构问题
现在整个 app 只有 HomeScreen，没有「项目内」的层级。真正做编辑器，需要想清楚导航栈：

HomeScreen（项目列表/搜索）
  └── ProjectScreen（项目根目录树 + 编辑器 Tab 栏）
        └── EditorScreen（代码编辑，单文件）

这三层的状态管理、导航、搜索上下文切换是整个 app 的骨架，要早点定下来。

你觉得哪个方向最想先做？ 我倾向从「数据层 + Room + 真实文件读写」开始打底，再做项目创建/导入 UX，这样后续所有功能都有真实数据跑。Wing chips 改成项目快捷入口这个视觉改动也可以很快搭进去。

核心问题重新定义
Wing chips 本质上是**「当前情境下最值得快速跳到的东西」**。这个「情境」其实有两种：

情境 A：主页，无项目打开
  → 最值得跳到的是 → 「我最近在做的几个项目」
情境 B：项目内部
  → 最值得跳到的是 → 「这个项目里我最近编辑的几个文件」

所以 Wing chips 不用改设计，只需要内容随情境切换——主页上两边都显示项目，进入项目后两边都变成该项目的最近文件。逻辑一致，设计不动。

项目列表放哪
现在主页底部有一大块空白（weight(1.4f) 那个插槽），闲置时什么都不显示，这正是项目列表天然的落脚点：
┌──────────────────────────────────┐
│                                  │
│    What are we building today?   │  ← hero greeting
│                                  │
│  [proj] [proj]  [ 🔍 Search ]  [proj] [proj]  │  ← wings 显示最近项目
│                                  │
│       > commands · # symbols     │  ← mode hints
│                                  │
│  ─────────────── RECENT ───────  │

│                                  │

整体主页状态机

① 闲置（无项目打开）
